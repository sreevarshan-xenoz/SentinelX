Overview

SentinelX Visualization Tools provide powerful visualization capabilities for network security monitoring, threat detection, and incident response. The visualization components include:

  • Network Graph Generator: Interactive network graphs showing connections between hosts
  • Alert Dashboard: Comprehensive dashboard for security alerts
  • Time Series Plotter: Visualize network traffic patterns over time
  • Heatmap Generator: Heatmaps showing traffic intensity across different network segments
  • GeoIP Mapper: Map showing geographic distribution of network connections
  • Report Generator: Generate comprehensive security reports

These tools are available through both a command-line interface (CLI) and a web-based dashboard application.

Installation

SentinelX can be installed using pip:

pip install sentinelx

Or by cloning the repository and installing from source:

git clone https://github.com/yourusername/sentinelx.git cd sentinelx pip install -e .

Dependencies

SentinelX requires the following dependencies:

  • Python 3.8+
  • NetworkX
  • Plotly
  • Dash
  • Folium
  • Pandas
  • NumPy
  • And more (see requirements.txt)

CLI Tool

The SentinelX CLI tool provides command-line access to all visualization capabilities.

Basic Usage

# Launch the CLI tool sentinelx-vis cli [command] [options] # Or using the module directly python -m src.visualization cli [command] [options]

Available Commands

Command Description Example
network Generate network graph visualization sentinelx-vis cli network --time-window 1h --format html --output network_graph.html
alerts Generate alert dashboard visualization sentinelx-vis cli alerts --severity high --format json --output alerts.json
timeseries Generate time series visualization sentinelx-vis cli timeseries --metric bytes --protocol tcp --format png --output traffic.png
heatmap Generate heatmap visualization sentinelx-vis cli heatmap --source-filter internal --format html --output heatmap.html
geomap Generate geographic IP map visualization sentinelx-vis cli geomap --country-filter US,CN,RU --format html --output geomap.html
report Generate comprehensive security report sentinelx-vis cli report --type detailed --format pdf --output security_report.pdf
dashboard Launch interactive web dashboard sentinelx-vis cli dashboard --port 8050

Common Options

Option Description Default
--time-window Time window for data (e.g., 1h, 24h, 7d) 24h
--format Output format (html, json, csv, png, pdf, md, svg) html
--output Output file path stdout
--config Path to configuration file ~/.sentinelx/config.json

Web Dashboard

The SentinelX Web Dashboard provides a user-friendly interface for accessing all visualization capabilities.

Starting the Dashboard

# Launch the web dashboard sentinelx-vis web [options] # Or using the module directly python -m src.visualization web [options]

Dashboard Options

Option Description Default
--port Port to run the web server on 8050
--host Host to run the web server on 127.0.0.1
--debug Run in debug mode False
--config Path to configuration file ~/.sentinelx/config.json

Dashboard Pages

The web dashboard includes the following pages:

  • Overview: Summary of security status and key metrics
  • Network: Interactive network graph visualization
  • Alerts: Alert dashboard with filtering capabilities
  • Time Series: Traffic patterns over time
  • Heatmap: Traffic intensity visualization
  • GeoIP: Geographic distribution of connections
  • Reports: Generate and view security reports
  • System Info: System status and configuration

API Reference

SentinelX provides a Python API for programmatic access to visualization capabilities.

Core Classes

VisualizationManager

Central manager for all visualization components.

from sentinelx.visualization import VisualizationManager # Initialize the visualization manager vis_manager = VisualizationManager() # Generate a network graph graph = vis_manager.generate_network_graph(time_window="1h") # Generate an alert dashboard dashboard = vis_manager.generate_alert_dashboard(severity="high")

VisualizationExporter

Handles exporting visualizations to various formats.

from sentinelx.visualization import VisualizationExporter # Initialize the exporter exporter = VisualizationExporter() # Export a visualization to a file exporter.export_visualization(visualization, format="html", output_path="output.html")

DashboardApp

Web dashboard application.

from sentinelx.visualization import DashboardApp # Initialize the dashboard app app = DashboardApp() # Run the dashboard app.run(host="127.0.0.1", port=8050, debug=False)

Examples

Generating a Network Graph

from sentinelx.visualization import VisualizationManager, VisualizationExporter # Initialize components vis_manager = VisualizationManager() exporter = VisualizationExporter() # Generate a network graph graph = vis_manager.generate_network_graph(time_window="1h") # Export to HTML exporter.export_visualization(graph, format="html", output_path="network_graph.html")

Creating a Custom Dashboard

import dash from dash import html, dcc import dash_bootstrap_components as dbc from sentinelx.visualization import VisualizationManager # Initialize visualization manager vis_manager = VisualizationManager() # Create a Dash app app = dash.Dash(__name__, external_stylesheets=[dbc.themes.BOOTSTRAP]) # Define layout app.layout = dbc.Container([ html.H1("Custom Security Dashboard"), dbc.Row([ dbc.Col([ html.H3("Network Activity"), dcc.Graph(id="network-graph") ], width=6), dbc.Col([ html.H3("Alert Summary"), dcc.Graph(id="alert-summary") ], width=6) ]) ]) # Define callbacks @app.callback( dash.dependencies.Output("network-graph", "figure"), dash.dependencies.Input("interval-component", "n_intervals") ) def update_network_graph(n): graph = vis_manager.generate_network_graph(time_window="1h") return graph.to_plotly_figure() # Run the app if __name__ == "__main__": app.run_server(debug=True)

Troubleshooting

Common Issues

Dashboard Not Starting

If the dashboard fails to start, check the following:

  • Ensure all dependencies are installed
  • Check if the specified port is already in use
  • Verify that the configuration file exists and is valid

Visualization Export Errors

If exporting visualizations fails:

  • Ensure the output directory exists and is writable
  • Check if the required dependencies for the specific format are installed
  • Verify that the visualization object is valid

Getting Help

If you encounter issues not covered here, you can:

  • Check the logs in ~/.sentinelx/logs/
  • Run with debug mode enabled: sentinelx-vis web --debug
  • Submit an issue on the GitHub repository
  • Contact support at support@sentinelx.example.com
